02The six things to learn first

Master these before chasing plugins.

Frames

Containers for screens, sections, and components. Frames control layout, clipping, constraints, and prototype behavior.

Auto layout

Direction, gap, padding, alignment, and sizing rules that adapt when content or container size changes.

Components

Reusable source objects. Copies become instances that inherit updates while preserving supported overrides.

Styles & variables

Named design decisions for color, typography, spacing, themes, modes, and developer-facing tokens.

Prototypes

Connections and interactions that simulate navigation, overlays, transitions, and state changes.

Developer handoff

Clear structure, measurements, assets, annotations, variables, and behavior that developers can inspect.

05Beginner traps

What makes a file feel messy.

Avoid

  • Grouping everything instead of framing it
  • Absolute-positioning every layer
  • Detaching instances for quick edits
  • Keeping names like “Frame 183”
  • Designing only one screen width
  • Using unnamed color values everywhere

Prefer

  • Explainable parent–child structure
  • Auto layout for related content
  • Component properties and variants
  • Clear, purpose-based names
  • Responsive checks at multiple widths
  • Semantic variables such as surface and accent

Your first exercise

Build one responsive profile card.

Use a vertical auto-layout card, a horizontal row for avatar plus identity, a reusable button component, and one color variable. Then resize the card and change the button label. If nothing breaks, the structure is doing its job.
